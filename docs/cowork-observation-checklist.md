# Cowork Observation Checklist

Use this while testing Cowork to capture UX patterns we should match or improve.

## Onboarding & Empty State

- What quick‑start prompts are shown?
- Where is the folder picker and how is scope explained?
- Does it encourage “plan first” or “review before run”?

## Task Creation

- How do you start a new task?
- How is a task named / titled?
- Can you queue multiple tasks easily?

## Progress / Steps

- How are steps displayed (timeline, checklist, progress bar)?
- Are steps user‑readable or raw command output?
- How does it show current action?

## Tool Logs / Command Visibility

- Are commands visible by default or collapsed?
- Is there a “show details” toggle?
- How much technical detail is shown to normies?

## Artifacts / Canvas

- Where do artifacts appear?
- What preview types are supported?
- Is HTML rendered in‑app or opened externally?
- Is any editing possible, or view‑only?

## Permissions / Approvals

- When does it ask for approval?
- What options are offered (allow once / always / deny)?
- How are approvals remembered across tasks?

## Scope & Context

- How is the selected folder displayed?
- Can you add/remove folders mid‑task?
- Is read‑only mode visible?

## Connectors / Web Access

- How are connectors enabled?
- How is web access described/warned?
- Is there a visible “network on/off” status?

## Task Lifecycle

- Can you pause or abort a task?
- What happens if the app closes or sleeps?
- Are tasks resumable?

## Performance / UX

- Time to first response
- How long long‑running tasks take
- Where it feels slow/confusing
